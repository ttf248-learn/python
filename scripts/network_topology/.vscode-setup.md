# VSCode 调试配置使用指南

## 📋 概述

本项目为 `process_network_topology.py` 脚本提供了完整的 VSCode 调试和开发环境配置。

## 🚀 快速开始

### 1. 打开项目
```bash
code /home/core/dev/python
```

### 2. 安装推荐扩展
VSCode 会自动提示安装推荐的扩展，或按 `Ctrl+Shift+P` 输入 "Extensions: Show Recommended Extensions"

### 3. 选择 Python 解释器
- 按 `Ctrl+Shift+P` 打开命令面板
- 输入 "Python: Select Interpreter"
- 选择 `python3` (推荐) 或系统的默认 Python

## 🔍 调试配置

### 预定义调试配置

在 VSCode 中按 `F5` 或 `Ctrl+Shift+D` 打开调试面板，选择以下配置：

#### 1. **Network Topology: 分析PID 1**
- 快速分析系统进程PID 1
- 只生成文本报告
- 适用于测试脚本是否正常工作

#### 2. **Network Topology: 分析指定PID**
- 交互式输入要分析的进程ID
- 生成所有格式的输出 (HTML, PNG, 文本)
- **推荐用于实际分析**

#### 3. **Network Topology: 生成HTML**
- 只生成交互式HTML拓扑图
- 适合需要详细查看网络关系时使用

#### 4. **Network Topology: 生成PNG**
- 只生成静态PNG图像
- 适合需要插入报告或文档时使用

#### 5. **Network Topology: 完整分析**
- 生成所有格式的输出
- 包含详细日志信息

### 使用步骤

1. 打开 `process_network_topology.py` 文件
2. 按 `F5` 启动调试
3. 在下拉菜单中选择配置
4. 如果选择需要输入的配置，会弹出提示框
5. 输入要分析的PID (如: 1, 1234, 5678)
6. 按回车开始调试

## ⚙️ 任务配置

### 可用任务

按 `Ctrl+Shift+P` 输入 "Tasks: Run Task"，选择以下任务：

#### 安装依赖
- 安装 `requirements.txt` 中的所有依赖
- 首次使用前必须执行

#### 运行脚本
- 快速执行脚本并查看输出
- 不会进入调试模式

#### 查看帮助
- 显示脚本帮助信息
- 查看所有可用参数

#### 检查网络拓扑目录
- 列出脚本目录的所有文件
- 确认文件完整性

#### 清理输出文件
- 删除 `output/` 目录中的所有文件
- 清理历史输出结果

## 🎯 开发技巧

### 设置断点
1. 在代码行号左侧点击设置断点
2. 或在代码行上按 `F9`
3. 调试时会自动在断点处暂停

### 条件断点
1. 右键断点
2. 选择 "Edit Breakpoint"
3. 输入条件表达式，如 `pid == 1`

### 日志点
1. 右键断点
2. 选择 "Edit Breakpoint"
3. 选择 "Log Message"
4. 输入要打印的文本，如 `Analyzing PID: {pid}`

### 调试变量
- 在调试面板中查看变量值
- 在调试控制台中输入变量名查看详情

## 🔧 自定义配置

### 修改默认PID
编辑 `.vscode/launch.json`:
```json
{
    "id": "pid",
    "default": "1",  // 修改此处的默认值
    "type": "promptString"
}
```

### 添加新的调试配置
在 `.vscode/launch.json` 的 `configurations` 数组中添加新配置：
```json
{
    "name": "配置名称",
    "type": "debugpy",
    "request": "launch",
    "program": "${workspaceFolder}/scripts/network_topology/process_network_topology.py",
    "args": ["-p", "1"],
    "console": "integratedTerminal"
}
```

## ❗ 注意事项

1. **权限要求**: 调试某些进程可能需要 root 权限
2. **Python环境**: 确保已安装所有依赖 (`pip install -r requirements.txt`)
3. **工作目录**: 调试配置会自动切换到脚本目录
4. **输出位置**: 默认输出到 `scripts/network_topology/output/`

## 🆘 常见问题

### Q: 提示 "Python interpreter not found"
**A**: 按 `Ctrl+Shift+P` → "Python: Select Interpreter" → 选择 `python3`

### Q: 调试时提示 "ModuleNotFoundError"
**A**: 执行 "安装network_topology" 任务安装依赖

### Q: 无法连接到远程进程
**A**: 检查是否有足够权限，或尝试用 `sudo` 运行 VSCode

### Q: 生成的图像无法打开
**A**: 确保有写入权限，输出目录可能需要手动创建
